# Pro Watermark

Apple ライクで、プライバシー第一の写真透かしツール。テキストやロゴの透かしを追加しながら、**元のメタデータとフル解像度を保持**します。すべてブラウザ内で完結し、画像がサーバーにアップロードされることは一切ありません。

[English](README.md) · [简体中文](README.zh-CN.md) · **日本語**

---

## 主な機能

- **テキスト / ロゴ透かし** —— テキストを入力するか、PNG/SVG ロゴをアップロード。
- **細かな調整** —— 9 つのプリセット位置に加え、自由なドラッグ・回転・サイズ・不透明度・色（プリセット + カスタムカラーピッカー）。
- **画像ごとの個別設定** —— バッチ内の各写真が独立した透かし状態を保持し、1 枚を編集しても他には影響しません。
- **バッチ処理** —— 複数の写真をまとめて処理。
- **iPhone の HEIC 対応** —— HEIC/HEIF 写真を自動でデコード。
- **メタデータを保持** —— 元の EXIF / ICC カラープロファイルを書き出し画像に無損失で復元。
- **カメラロールの先頭に並ぶ** —— 書き出し時に EXIF の撮影時刻を「今」に更新するため、透かし入りの写真が最新位置に並びます。さらに向き（Orientation）タグを正規化し、端末による二重回転を防ぎます。
- **アルバムに直接保存** —— モバイルではネイティブの共有シートから「画像を保存」でアルバムへ、デスクトップではダウンロード。
- **視覚的に無損失な書き出し** —— フル解像度・JPEG 品質 0.95・黒帯なし。
- **多言語対応** —— English / 简体中文 / 日本語。
- **洗練されたモーションと UI** —— スライドするタブ／位置ピル、キューのスムーズな追加・削除、オーロラ風の背景、丁寧な書き出しプログレスリング、そして画面が真っ白にならないエラーバウンダリ。

> **「無損失」について：** キャンバス上で透かしを合成するツールは必ず一度画像を再エンコードします。そのため本ツールは **メタデータ無損失 + 視覚的に無損失（品質 0.95）** であり、バイト単位で同一ではありません。元の EXIF/ICC は保持され、解像度が下がることもありません。

## 仕組み

すべて**クライアントサイド**で動作します。写真をデコードし、キャンバスに透かしを描画して JPEG に再エンコード。その後、元ファイルのメタデータ領域（EXIF/ICC）をバイナリレベルで新しい JPEG に精密に縫い戻し、撮影時刻と向きだけを調整します。画像が**端末から出ることはありません**——写真に人物（生徒など）が写っている場合、これは大きな利点です。

## 技術スタック

- **React 18** + **TypeScript**
- **Rspack**（SWC）によるビルド
- **Fabric.js v6** によるキャンバスエディタ
- **Effect** による画像処理パイプライン
- **Tailwind CSS** によるスタイリング
- **Framer Motion** によるアニメーション、**Vaul** によるモバイルドロワー
- **i18next** によるローカライズ、**lucide-react** によるアイコン
- **heic-to** による HEIC デコード
- **Vitest** によるテスト、**Biome** による lint/フォーマット
- **Vercel** によるホスティング（Analytics と Speed Insights を含む）

## はじめに

```bash
npm install
npm run dev      # 開発サーバーを http://localhost:8080 で起動
```

### スクリプト

| コマンド | 説明 |
| --- | --- |
| `npm run dev` | 開発サーバーを起動 |
| `npm run build` | `dist/` への本番ビルド |
| `npm run test` | ユニットテスト（Vitest）を実行 |
| `npm run lint` | Biome で lint |
| `npm run format` | Biome でフォーマット |
| `npm run check` | Biome の lint + フォーマットチェック |

## プロジェクト構成

Feature-Sliced Design を採用：

```
src/
  app/        アプリのエントリ、ルート状態、グローバルスタイル
  features/   キャンバスエディタとエディタレイアウト
  entities/   透かしのドメイン型とジオメトリ
  kernel/     バイナリのメタデータ処理と処理パイプライン
  shared/     i18n、共有 UI（エラーバウンダリ）
```

## デプロイ

Vercel でホスティングしており、`main` への push ごとに自動デプロイされます。Web Analytics と Speed Insights は Vercel のダッシュボードで一度有効化する必要があります。
